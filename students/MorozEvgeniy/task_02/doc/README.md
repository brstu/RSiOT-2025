## Цель работы

Целью лабораторной работы является получение практических навыков
развёртывания stateless-приложения в среде Kubernetes с использованием
контейнеризации Docker, а также настройка механизмов отказоустойчивости
и мониторинга состояния контейнеров.

---

## Описание проекта

В рамках лабораторной работы был разработан HTTP-сервис на языке Go.
Приложение выводит информацию о студенте и счётчик посещений страницы.
Счётчик хранится во внешнем хранилище Redis.

Приложение упаковано в Docker-контейнер и развёрнуто в Kubernetes-кластере
с использованием следующих ресурсов:
- Namespace
- Deployment
- Service
- Ingress
- ConfigMap

Для контроля состояния контейнеров реализованы эндпоинты
`/live` и `/ready`, используемые Kubernetes probes.

## Этапы выполнения работы

1. Разработка HTTP-сервиса на языке Go.
2. Создание Dockerfile и сборка Docker-образа приложения.
3. Публикация Docker-образа в Docker Hub.
4. Развёртывание локального Kubernetes-кластера с помощью Minikube.
5. Применение Kubernetes-манифестов (Namespace, Deployment, Service, Ingress).
6. Развёртывание Redis в Kubernetes.
7. Проверка состояния pod'ов и доступности сервиса.

## Проверка работоспособности

Проверка выполнялась с помощью HTTP-запросов к сервису:

```bash
curl http://127.0.0.1:<port>/
curl http://127.0.0.1:<port>/live
curl http://127.0.0.1:<port>/ready
```

В результате:
- основной эндпоинт возвращает информацию о студенте и номер посещения;
- эндпоинт `/live` возвращает статус `LIVE`;
- эндпоинт `/ready` возвращает статус `READY`.
